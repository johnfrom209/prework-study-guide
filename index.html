HTML

  • The head element contains information about the webpage.
  • The head contains the links to other files such as .css or .js for styling and javascript.
  • The body element represents the visible content shown to the user.
  • Keep things nice and organized within their own sections.

CSS

  • Cascading Style Sheet
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside the element.
  • Use either hexadecimal or names to pick colors. Hexadecimal is more flexable with adjustments.
  • Avoid repeating code by grouping mutiple classes together.
  • Styling can be done internally, inline, or externally.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A commits all files in the working branch.
  • git commit -m "abc" links a message to the commit. Keep this short and descriptive.
  • A Pull Request is usualy reviewed before merging to the main/master branch.
  • Create a new branch, off of the main, for each new component you plan to add.

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Use console.log to squash those bugs.
  • The js script needs to be called in the body instead of the Head.
